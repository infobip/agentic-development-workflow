# Autonomous Implementation Instructions

These instructions are the canonical rules for the optional autonomous implementation kit. Tool-specific adapters such as `CLAUDE.md` must defer to this file.

## Purpose

This directory contains an optional implementation mode for the Implementation phase of the workflow. The loop starts a fresh coding-agent session for each task, completes exactly one task, records the result, commits, and exits.

Use this only after Research, Planning, and Task Definition are complete.

Expected artifact flow:

```text
RESEARCH.md -> PLAN.md -> TASKS.md -> ACTIVITY.md + commits
```

## Required files

- `README.md` — usage and adapter contract for the loop driver.
- `AGENTS.md` — canonical instructions for loop sessions.
- `PROMPT.md` — per-session prompt loaded by the loop driver.
- `TASKS.md` — task registry generated by `/to-tasks`.
- `ACTIVITY.md` — append-only log of each session.
- `PATTERNS.md` — human-curated corrections for recurring agent mistakes.
- `SANDBOX.md` — isolation and review guidance.
- Git history — continuity between stateless sessions.

## Loop rules

Each fresh session must:

1. Read `AGENTS.md`, `TASKS.md`, `ACTIVITY.md`, `PATTERNS.md`, relevant code, and git history.
2. Select the first task in `TASKS.md` with a task-status line `- **Passes:** false`.
3. Complete exactly that task.
4. Stay inside task scope.
5. Run the relevant verification commands from the project instructions.
6. Append an entry to `ACTIVITY.md` whether the task succeeds or fails.
7. Mark `**Passes:** true` only after all required checks pass.
8. Commit all changes, including partial progress and the activity log.
9. Stop without starting another task.

## Failure protocol

If the task cannot be completed:

1. Record what failed in `ACTIVITY.md` under **Blockers**.
2. Record what was tried and what the next session should try.
3. Leave `**Passes:** false`.
4. Commit the current state.
5. Stop.

## Loop driver

`loop.sh` is agent-agnostic. Practitioners configure the fresh-session command through `LOOP_AGENT_COMMAND` or by editing `DEFAULT_AGENT_COMMAND` in a local copy of the script. Tool-specific adapter files such as `CLAUDE.md` may provide concrete command examples, but this file remains canonical.

## Human responsibilities

- Run on a dedicated branch or worktree.
- Configure `LOOP_AGENT_COMMAND` for the chosen coding agent.
- Review generated commits before merging.
- Add short corrections to `PATTERNS.md` when sessions repeat the same mistake.
- Keep project instructions concise because they are loaded into every session.

## Safety

- Do not read or commit secrets, tokens, private keys, credential stores, browser session data, `.env*`, `.secrets`, or `secrets/**`.
- Do not run destructive commands unless the task explicitly requires them and the human has approved.
- Do not modify unrelated files.
- Do not hide failures by marking tasks complete without verification.
